# Report Structure

The visual structure and components of pytest-aitest HTML reports.

## Design Philosophy

Reports answer one question: **"Which configuration should I deploy?"**

Every visual element supports this goal through:

1. **Progressive disclosure** ‚Äî Summary first, details on demand
2. **Comparison-first** ‚Äî Winner highlighting, sorted rankings
3. **Scalability** ‚Äî Works for 2 agents or 20 agents
4. **Actionable insights** ‚Äî Not just metrics, but what to fix

## Adaptive Behavior

The report layout adapts based on what was tested:

| Scenario | Agent Leaderboard | Detailed Results |
|----------|-------------------|------------------|
| 1 agent, 1 test | ‚ùå | ‚úÖ Expanded |
| 1 agent, N tests | ‚ùå | ‚úÖ Expanded |
| N agents, same tests | ‚úÖ | ‚úÖ Grouped by test |
| Session tests | ‚ùå (unless multi-agent) | ‚úÖ Session grouping |

### Detection Logic

```python
if len(unique_agents) == 1:
    # Simple mode: no comparison views
    show_agent_leaderboard = False
elif len(unique_agents) > 1:
    # Comparison mode
    show_agent_leaderboard = True
```

## Scalability Requirements

The design MUST work at these scales:

| Scale | Behavior |
|-------|----------|
| 2 agents | Leaderboard shows 2 rows |
| 8 agents | Leaderboard with 8 rows, vertical scroll |
| 20 agents | Leaderboard with pagination |
| 50+ tests | Tests expanded by default, browser scroll |

### Anti-Patterns (What NOT to Do)

‚ùå **Don't** show side-by-side cards that shrink with more items  
‚ùå **Don't** truncate agent names ‚Äî wrap or tooltip instead  
‚ùå **Don't** show tiny unreadable diagrams  
‚ùå **Don't** require horizontal scrolling for core content  
‚ùå **Don't** duplicate information across multiple views

## Key Principles

1. **One source of truth** ‚Äî Agent Leaderboard is THE comparison view
2. **AI explains, templates display** ‚Äî AI writes insights in markdown
3. **Sessions are grouping, not special** ‚Äî Same test cards, visual connectors
4. **Progressive disclosure** ‚Äî Click to expand details
5. **No redundancy** ‚Äî Each piece of information appears once
